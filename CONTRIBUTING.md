# Contributing to Security Pulse

Thank you for your interest in contributing! This is a learning project and we welcome contributions of all kinds.

## Ways to Contribute

### 🔗 Suggest a New Feed

1. Open an [Issue](https://github.com/yourusername/security-pulse/issues) with the title "Add Feed: [Feed Name]"
2. Include:
   - Feed name
   - RSS feed URL
   - Why this feed is valuable for security professionals
   - Example: Any recent articles that would be captured

### 🐛 Report a Bug

1. Open an Issue with a clear title describing the problem
2. Include:
   - What you expected to happen
   - What actually happened
   - Steps to reproduce
   - Any error messages or logs

### 💡 Suggest Improvements

- Better output formatting
- Performance improvements
- New features (filtering, tags, etc.)
- Documentation improvements

### 💻 Code Contributions

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-feature`
3. Make your changes
4. Test locally: `python pulse.py`
5. Commit with clear messages: `git commit -m "Add feature: description"`
6. Push to your fork: `git push origin feature/your-feature`
7. Open a Pull Request with a description of your changes

## Development Guidelines

- Keep code simple and readable
- Add comments for complex logic
- Test changes locally before submitting
- Follow PEP 8 style guide
- Update README.md if adding new features

## Questions?

Feel free to open a Discussion or Issue if you have questions!

---

**All contributions are appreciated, no matter how small!**
